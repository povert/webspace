- 本地个人数据分析工作台

**版本号**: V2.0
**状态**: 已实现
**技术栈**: Python Flask + HTML5/JS (Vanilla)
**核心定位**: 一款面向个人的、基于文件系统的本地 Web 工具。通过"文件系统即数据库"的理念，结合标准化的脚本协议，实现数据浏览、文件管理、脚本执行、进度监控及报表生成的闭环管理。无需登录，开箱即用。

---

## 1. 项目背景与目标

### 1.1 背景
个人开发者或数据分析师在日常工作中，经常需要处理格式各异的数据（JSON, Log, CSV 等）。传统的做法是在本地编辑器写代码、在终端运行、在浏览器查看结果，流程割裂且难以管理版本。用户需要一个轻量级的本地 Web 服务，能够像文件管理器一样直观地查看和管理数据/脚本/报表，并能一键执行脚本生成可视化报表。

### 1.2 目标
1. **零配置本地运行**: 无需数据库，无需登录认证，启动即可使用。
2. **多工作空间管理**: 支持创建、切换、重命名、删除多个独立工作空间，每个空间拥有独立的 data/、scripts/、reports/ 目录。
3. **完整文件管理**: 支持文件/文件夹的浏览、创建、重命名、删除，以及拖拽移动。
4. **实时文件系统映射**: Web 界面实时反映磁盘上的文件变化，支持外部文件拖入后刷新可见。
5. **灵活的脚本编辑**: 脚本文件支持在线编辑，方便快速迭代。
6. **标准化脚本执行**: 通过解析脚本元数据，动态生成参数表单；通过标准输出协议，实时展示执行进度。
7. **报表资产追溯**: 自动关联报表与生成它的脚本、数据及参数，支持一键重跑。

## 2. 技术栈说明

*   **后端**: **Python Flask**
    *   利用 `os`, `shutil`, `subprocess` 模块进行文件系统操作和进程管理。
    *   使用 Flask 原生 `StreamingResponse` + `stream_with_context` 实现 SSE 服务器推送事件，用于实时日志传输。
    *   单文件后端 `app.py`，所有路由集中管理，无蓝图、无 ORM。
*   **前端**: **HTML5 + Vanilla JavaScript**
    *   保持极简，无复杂构建步骤，无框架依赖。
    *   使用 `fetch API` 进行异步请求。
    *   使用 CSS Grid/Flexbox 实现响应式布局，暗色主题。
    *   使用 HTML5 Drag & Drop API 实现文件拖拽移动。
*   **数据存储**: **本地文件系统**
    *   所有状态（数据、代码、报表、元数据）均持久化存储在本地文件夹中。

### 5.2 进度上报协议 (Progress Protocol)
脚本需向 `stdout` 打印特定 JSON 字符串，系统会自动捕获并解析。
*   **进度更新**: `print('{"progress": 0.5, "msg": "正在处理数据..."}')`
*   **完成信号**: `print('{"status": "success", "report": "report_123.html"}')`
*   **错误信号**: `print('{"status": "error", "msg": "文件未找到"}')`

### 5.3 参数接收与输出
*   **参数接收**: 推荐使用 `argparse` 库接收命令行参数。
*   **输出路径**: 系统会强制传入 `--output` 参数，脚本必须将生成的 HTML 写入该路径。

## 6. 非功能性需求

### 6.1 性能与体验
*   **秒开**: 页面加载时间 < 1秒。
*   **流畅拖拽**: 拖拽移动文件即时响应，视觉反馈 < 50ms。
*   **低延迟日志**: 脚本输出日志到前端显示的延迟 < 500ms。
*   **大文件处理**: 预览时 5MB 以上文件截断显示前 1MB，避免卡顿。

### 6.2 安全性 (本地环境)
*   **路径沙箱**: `safe_path()` 函数验证所有路径在 workspace 目录内，防止目录遍历攻击。
*   **超时保护**: 脚本执行默认超时 60 秒，防止死循环卡死服务。
*   **进程隔离**: 脚本以子进程方式运行，`cwd` 锁定在工作空间目录。
*   **数据保护**: 只能通过 API 删除文件，防止误操作；删除前有确认弹窗。

## 8. 用户操作流程

1. **启动与访问**: 用户在终端运行 `python app.py`，浏览器打开 `localhost:5120`。
2. **管理工作空间**: 通过 Header 下拉或导航栏底部"工作空间管理"创建/切换/重命名/删除工作空间。
3. **准备数据**: 用户将数据文件拖入工作空间的 `data/` 文件夹，或在 Web 界面通过"新建文件"创建。
4. **浏览与管理**:
   - 在 Web 界面点击"数据"查看文件列表，网格/列表视图切换。
   - 创建子文件夹整理文件，拖拽移动文件到不同目录。
   - 点击文件名弹出只读预览窗口，CSV 以表格展示，JSON 格式化显示。
5. **编写/编辑脚本**:
   - 点击"脚本"切换到脚本目录，双击文件夹进入子目录。
   - 点击脚本旁的"编辑"按钮，在代码编辑器中修改。
   - 或通过"新建文件"创建新的 `.py` 脚本。
6. **配置与执行**:
   - 点击脚本旁的"执行"按钮。
   - 系统弹出参数框，文件类型参数自动列出 data/ 目录下文件供选择。
   - 点击"开始运行"。
7. **监控进度**:
   - 底部控制台弹出，进度条随脚本执行逐渐前进。
   - 日志区实时显示 stdout/stderr 输出。
   - 可随时点击"停止"终止运行。
8. **查看报表**:
   - 执行成功后，切换到"报表"页面。
   - 点击 HTML 报表卡片在新标签页预览。
   - 点击"详情"查看该报表的脚本、数据源、参数关联。
   - 点击"重跑"自动填充上次参数重新执行。
